# BBQ Clear 游戏迭代设计文档 (2026-04-01)

## 1. 概述
本次迭代旨在优化游戏体验、修复动画错误、增强核心玩法逻辑，并引入局域网排行榜系统，增强社交互动。

## 2. 交互与动画优化
### 2.1 彻底移除音效
*   **动作**：移除 `App.jsx` 中的 `playSound` 函数、`audios` 配置对象以及所有 `playSound` 调用点。
*   **影响**：游戏将进入静音模式，减少外部资源依赖和加载时间。

### 2.2 修复并重定向“上菜”动画
*   **现状**：目前动画使用固定的位移 `x: 300, y: -400`，表现不稳定且有时由于容器限制显示异常。
*   **优化**：
    *   在 `gameAnimations.js` 中重构 `playServeAnimation`。
    *   使用 GSAP 的 `to` 动画，将 `x` 和 `y` 设置为相对于**排行榜按钮**在屏幕上的坐标。
    *   动画路径：缩小 -> 匀加速向按钮位置飞走 -> 消失。
*   **按钮位置获取**：在 `App.jsx` 中通过 `useRef` 获取排行榜按钮的 DOM 引用，并传递给 `Grill` 及其动画逻辑。

### 2.3 选中态逻辑修复
*   **逻辑**：在 `App.jsx` 的 `handleDragStart` 函数中添加 `setSelectedSkewer(null)`。
*   **效果**：一旦用户开始“拖拽”任何物品，当前的“点击选中”状态将立即重置。

## 3. 核心逻辑改进
### 3.1 补给随机化
*   **修改点**：`useGameState.js` 中的 `initGame` 和 `refillSpecificGrill`。
*   **实现**：使用随机索引从 `null` slots 中选择填入位置，而不是顺序填入 0/1。

### 3.2 动态补给加速
*   **条件**：在 `completeServe` 后，检查 `totalVisibleCount < 20` (所有 12 个烤架上当前可见的菜品总和)。
*   **表现**：若满足条件，空位补足数量改为 2 个（50% 概率）或 3 个（50% 概率），增加场面肉量。

## 4. 局域网排行榜系统
### 4.1 后端服务 (Node.js/Express)
*   **功能**：提供 `/api/scores` 接口（GET 获取 Top 10，POST 提交新分数）。
*   **存储**：保存在根目录下的 `server/scores.json`。
*   **部署**：在同一个父目录下运行一个简单的脚本，Vite 通过 `proxy` 转发。

### 4.2 排行榜 UI
*   **形式**：右侧滑出的抽屉式面板 (`LeaderboardDrawer`)。
*   **交互**：点击主界面的“排行榜”图标触发。打开时，**暂停倒计时**且**禁用所有游戏交互**。

### 4.3 终局流程
*   游戏结束 -> 显示得分 -> 弹出名字输入框 -> 点击提交后，名字和分数发送至后端。

## 5. 开发步骤
1.  **环境配置**：安装 `express`, `cors` 依赖，配置 `vite.config.js` 代理。
2.  **后端实现**：编写轻量级 Express 服务器供应。
3.  **核心逻辑重构**：修改 `useGameState.js`。
4.  **UI 开发**：创建 `LeaderboardDrawer` 组件。
5.  **动画优化**：重写 `gameAnimations.js`。
6.  **清理与集成**：移除音效并整合逻辑。
