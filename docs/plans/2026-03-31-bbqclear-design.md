# BBQ Clear 游戏设计文档

> **项目名称**: bbqclear (网页端消除小游戏)
> **日期**: 2026-03-31
> **技术栈**: Vite + React + GSAP (动画库) + Vanilla CSS (样式)

---

## 1. 游戏概述 (Overview)
**bbqclear** 是一款基于烧烤场景的 3-Match (三消) 策略游戏。玩家需要在有限的时间内，通过在烧烤架之间调度肉串，将相同的 3 串食材凑齐并“上菜”，从而清理所有食材并获得高分。

## 2. 视觉与美学设计 (Visual & Aesthetics)
### 2.1 调色板 (Color Palette)
- **背景**: 深焦糖色 (Deep Burnt Brown) / 碳黑色 (Charcoal Black)，带有微弱的皮革或木质纹理。
- **烧烤架**: 金属质感 (Metallic Grey)，缝隙透出暗红色的“炭火”微光。
- **计时器**: 琥珀色 (Amber)，剩余 10 秒时转为亮红色并伴随跳动。
- **得分**: 金色 (Gold)，伴随缩放动画。

### 2.2 食物资源 (Food Assets)
- 使用具有立体感和高细节度的图标：肉串、玉米、香菇、鸡翅、大虾。
- 选定状态下带有 1.1x 放大及黄色外发光效果。

---

## 3. 核心游戏逻辑 (Core Mechanics)
### 3.1 网格系统 (Grid System)
- **初始规模**: 12 个烧烤架 (3x4 布局)。
- **单架容量**: 每个烤架拥有 3 个插槽 (Slots)。
- **补位机制**: 每个架子下方有一个隐藏的“待处理序列”(Pending Stack)，当前排位置腾空时，序列中的肉串自动向上补位。

### 3.2 交互逻辑 (Interaction)
- **拖拽放置 (Drag & Drop)**: 
  - 玩家可以从任何有空位的架子中提取肉串，并拖动到另一个未满的架子上。
  - 单个架子若已满 (3 串不同类型)，则无法再移入。
- **自动上菜 (Auto-Match)**:
  - 当一个架子内出现 3 串完全相同的食材时，触发“上菜”动画，食材消失并得分。

### 3.3 胜负与得分 (Scoring & Win/Loss)
- **胜利**: 在时间结束前清理完所有预设的食材。
- **得分项**:
  - 单次清理: +100。
  - 时间奖励: 剩余秒数 × 10。
- **失败**: 倒计时结束，仅保留已清理食材的基础积分。

## 4. 动画表现 (Animations)
使用 **GSAP** 实现高频率的视觉反馈：
- **上菜动画**: 缩放 -> 斜向滑出 -> 烟雾粒子效果。
- **补位平移**: 线性或带有轻微回弹的 Y 轴平移。
- **拖拽提示**: 悬停在合法网格上时的缩放或颜色高亮变化。

## 5. 开发路线图 (Roadmap)
1. **Phase 1**: 环境搭建与基础数据模型 (Grill, Slot, Food)。
2. **Phase 2**: 拖拽交互逻辑 (Drag and Event Handlers)。
3. **Phase 3**: 匹配消除与补位系统。
4. **Phase 5**: 视觉与 GSAP 补全。
5. **Phase 6**: 倒计时与排行榜持久化。
