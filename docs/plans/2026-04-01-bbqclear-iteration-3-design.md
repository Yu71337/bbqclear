# BBQ Clear 游戏迭代第 3 版设计文档 (2026-04-01)

## 1. 概述
本次迭代引入核心玩法策略——“烤熟度”与“烧焦”逻辑，同时修复多项交互 Bug 并优化服务器监控。

## 2. 交互与动画优化 (Bug 修复)
### 2.1 倒计时恢复
*   **动作**：修复 `useGameState` 中计时器 Effect 的依赖项。确保当 `isPaused` 从 `true` 变为 `false` 时（排行榜收起），计时器能自动重启。

### 2.2 拖放交互稳定性 (解决卡死)
*   **分析**：GSAP 动画可能在动画进行中锁定了某些属性或 `pointer-events`。
*   **优化**：
    *   在 `Skewer.jsx` 入场动画中显式设置 `pointer-events: none` 并在完成时恢复。
    *   优化 `onDragStart` 获取焦点的逻辑，确保即使在动画中也能被捕获。

### 2.3 “上菜”动画回归
*   **动作**：将 `playServeAnimation` 的位移逻辑从动态目标还原为固定的屏外右上角位移（如 `x: 500, y: -500`）。
*   **目的**：回归经典版本的视觉冲击感。

## 3. 烤熟度与烧焦系统 (核心玩法)
### 3.1 熟度逻辑
*   **状态模型**：物品新增 `level` (0-10)。
*   **随时间增加**：在 `useGameState` 中开启一个 5 秒周期的 `donenessEffect`。所有可见且未烧焦的食材 `level + 1`。
*   **上菜阈值**：只有三连食材中每个的 `level >= 1` 时才触发 `completeServe`。
*   **降温机制**：当 `level > 5` 的食材被移动到其他盘子时，重置其 `level` 为 3。

### 3.2 烧焦逻辑
*   **判定**：`level === 10` 时变为“烧焦”状态。
*   **表现**：`Skewer.jsx` 背景变黑，产生黑烟特效。
*   **规则**：烧焦物品无法参与“上菜”匹配，但可以自由移动。

### 3.3 烟雾特效 (视觉)
*   **青烟**：根据 `level` (1-9) 动态决定 CSS 烟雾粒子数量。
*   **黑烟**：烧焦后显示。
*   **运动**：特效与 `Skewer` 绑定，随拖拽和动画一起平移。

## 4. 智能保底与终局判定
### 4.1 智能补给 (方案 B)
*   **逻辑**：在 `completeServe` 触发新补位时，先计算场上可见且未烧焦的各品种数量。
*   **保底**：如果场上没有任何同品种 3 连的可能性（即使全场加起来也没有），强制从 `pending` 中挑选全场已存最多的那个品种来补齐至 3 个。

### 4.2 特殊失败判定
*   **逻辑**：由于“烤焦”占位，若场上空位不足以生成 3 个同种正常食材，且现有可见食材已无法凑出 3 连，触发失败弹窗“已没有更多食材能上菜”。
